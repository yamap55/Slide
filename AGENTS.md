# AI エージェント向けガイドライン

このリポジトリは reveal.js を利用したスライド管理システムです。AI エージェントがスライドを作成する際のガイドラインを以下に示します。

## リポジトリ概要

- **目的**: reveal.js を使用したスライドの管理・配信
- **デプロイ**: GitHub Pages で公開
- **URL 形式**: `http://yamap55.github.io/Slide/index.html?slide=YYYYMMDD/filename.md`

## スライド作成ガイドライン

### 作成プロセス

1. **事前確認**: 作成前に質問や疑問を洗い出し、ユーザーに確認する
2. **計画立案**: 予定を建ててから実行する
3. **不明点の扱い**: 作成中にわからない部分は「不明」「調査が必要」などの文言で明記する
4. **完成度**: 完成品ではなく、方向性が合っているかユーザーに確認しながら進める

### スライド構成の方針

- **時間目安**:
  - スライドそのまま話すとして **15 分程度** を目安
  - 雑談や口頭での補足を含めて **30 分程度** で実施する予定
- **詳細の扱い**:
  - 発表内容によるが基本的には時間内で全てを説明することは難しいため、詳細については割愛することを検討する
  - あくまでも「**イメージを掴んでもらうこと**」が目的
  - 詳細なルールを知ってもらう・覚えてもらうことは目的ではない
  - 細かいルール説明よりも理解を促す構成にする

### ディレクトリ構造

- スライドは `YYYYMMDD/` 形式のディレクトリに配置
- 例: `20241225/slide.md`, `20241225/my_presentation.md`

### ファイル命名

- ファイル名に特別な制約はありません
- GitHub Pages の URL として使用されるため、適切な名前を選択してください
- 推奨: `slide.md` または内容を表す分かりやすい名前

### README.md への追加

スライドを作成したら、README.md のスライド一覧の先頭行に追加します。

#### README.md の構造

README.md の冒頭には以下のような表形式でスライド一覧があります：

```markdown
| タイトル | 日付 | コード | 備考 |
| -------- | ---- | ------ | ---- |

|
| [スライドタイトル](URL) | YYYY/MM/DD | [src](GitHubへのリンク) | 備考 |
```

#### 追加方法

1. スライドの日付で確認し、日付の降順（新しい順）で配置する
2. 上記の形式に従って 1 行を追加
3. タイトル、日付、コード（ソース）、備考を記載

#### 例

```
| [Dockerfileのlinterについて](http://yamap55.github.io/Slide/index.html?slide=20240808/slide.md) | 2024/08/08 | [src](https://github.com/yamap55/Slide/blob/main/20240808/slide.md) | LT |
```

#### テンプレート

README.md の末尾にテンプレートがあります。これをコピーして使用してください：

```markdown
| [](http://yamap55.github.io/Slide/index.html?slide=) | // | [src](https://github.com/yamap55/Slide/blob/main/) | |
```

```markdown
# スライドタイトル

---

## セクションタイトル

--

## サブセクション

内容...

---

## 次のセクション

内容...
```

#### reveal.js の区切り文字

- **水平区切り**: `---` (新しいスライド)
- **垂直区切り**: `--` (同じスライド内の縦方向の区切り)

#### スタイル指定

- 各スライドファイルの先頭に上記の CSS を配置
- タイトルの大文字変換を無効化

### 内容について

- テーマに制約はありません
- IT エンジニア向けの内容が多いですが、一般的な内容も含まれます
- 各スライドは独立した内容として作成してください
- 他のスライドとの関連性を考慮する必要はありません

### 画像ファイル

- 画像は同じディレクトリ内に配置
- 相対パスで参照: `![説明](image.png)`
- 画像ファイルの命名に特別な制約はありません
- **例**:
  - 画像ファイル: `20240808/image.png`
  - スライドファイル: `20240808/slide.md`
  - スライド内の参照: `![説明](./image.png)` または `![説明](image.png)`
- 異なる形式も可能: `20231221/2023-12-10-14-10-26.png`（日付を含むファイル名）

### 既存スライドの参考

- リポジトリ内の既存スライドを参考にしてください
- 特に `20240808/slide.md` など、最近のスライドを参照
- README.md に実際のスライドへのリンクと一覧があります
- スライドの構造、構成、スタイルなどを実際のファイルで確認できます

#### スライドスタイルの特徴

既存のスライドを見ると以下のような特徴があります：

- **タイトルスライド**: H1 でタイトルを記載
- **アジェンダ**: セクション構成を明確に示す
- **セクション分け**: 主要トピックごとに`---`で区切る
- **サブセクション**: 詳細説明は`--`で縦方向に展開
- **コードブロック**: プレゼンテーション用に簡潔に
- **画像**: 相対パスで参照、説明文をつける

## 技術仕様

### reveal.js バージョン

- 使用バージョン: 3.5.0
- CDN から読み込み

### 対応機能

- Markdown サポート
- コードハイライト (zenburn テーマ)
- スピーカーノート
- PDF 出力対応

#### スピーカーノートの追加方法

スピーカーノートはスライドの内容の後に `???` を使って追加できます。

```markdown
## スライドタイトル

スライドの内容...

???

ここにスピーカーノートを記述します。
発表者向けの補足情報や話す内容をここに記載します。
```

### PDF 出力方法

- URL 末尾に `&print-pdf` を追加
  - 例: `http://yamap55.github.io/Slide/index.html?slide=20240808/slide.md&print-pdf`
- ブラウザの印刷機能で PDF 保存
- 印刷設定で「背景のグラフィック」にチェックを入れる
- 向きが縦向きの場合、横方向に変更する

### GitHub Pages へのデプロイ

- `main` ブランチのルートディレクトリが GitHub Pages として公開されます
- 公開 URL は自動的に `https://yamap55.github.io/Slide/index.html?slide=YYYYMMDD/filename.md` になります
- 新しいスライドを追加する場合は、`main` ブランチに push するだけで自動的に公開されます

## 注意事項

### 作成時の必須チェック項目

1. **CSS スタイル**: 必ずスライドファイルの先頭に CSS スタイルを配置（タイトルの大文字変換を無効化）
2. **区切り文字**: reveal.js の区切り文字を正しく使用
   - `---`: 新しいスライド（水平方向）
   - `--`: 同じスライド内の縦方向の区切り
3. **独立した内容**: 各スライドは独立した内容として作成
   - 他のスライドの補足説明を前提としない
   - 自己完結的な内容にする

### ファイル命名・URL

4. **ファイル名**: GitHub Pages の URL として使用されるため適切に命名
   - 推奨: 英数字とハイフンやアンダースコアのみ使用
   - スペースや特殊文字は避ける（URL エンコーディングされるため）
5. **ディレクトリ**: `YYYYMMDD/` 形式のディレクトリに配置する
6. **README.md への追加**: スライド作成後、README.md のスライド一覧に追加する必要があります

### 画像

7. **画像パス**: 相対パスで画像を参照
8. **画像配置**: 画像はスライドファイルと同じディレクトリに配置
